This page is related to deploying static website to public Internet using GitHub pages

(the basic scheme for deploying is here)

Note that this is handy tool Jekyll used to generate static sites

Let's see how to deploy website step by step:

  1. First of all we need content

    2. We can create site content from scratch (like this website) or use special tools (like Jekyll)

  2. Once we created a content we can publish it using GitHub Pages

    3. GitHub Pages is a service offered by GitHub. Specifically, GitHub Pages are public webpages that are hosted and published through GitHub.

    Why GitHub Pages? GitHub Pages offers extensive integration and support for Jekyll. By using both, you'll benefit from:

    • Easy setup
    • Troubleshooting your site
    • Updating and maintaining your site

    Note: Remember, it is possible to follow all of the steps with your own content â€” just make sure that your HTML is inside of a file called index.html (a GitHub Pages requirement).

  3. Make repository on GitHub and push your content to it

    4. Specifically, the repo's name must be in the following format: your-user-name.github.io

  4. Now you can access your website on URL like this: https://your-user-name.github.io/
